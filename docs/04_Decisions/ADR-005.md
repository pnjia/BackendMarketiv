# ADR-005 — Simpan Counter Denormalisasi di `campaigns`

## Status

Diterima.

## Context

- Dashboard UMKM dan job board sering menampilkan agregat campaign: total klaim, dana terpakai, sisa budget.
- Menghitung nilai-nilai ini dari koleksi lain (claims, transactions) setiap kali render mahal dan lambat.

## Decision

Simpan data denormalisasi langsung pada `campaigns`: `totalClaims`, `spentAmount`, `remainingBudget`. Nilai diperbarui oleh proses/Function saat event terkait terjadi, meski secara teknis dapat diturunkan dari koleksi lain.

## Consequences

- (+) Dashboard dan job board cepat tanpa agregasi mahal.
- (−) Ada risiko ketidaksinkronan; nilai harus dijaga konsisten lewat update terkontrol (Functions/service).
- (−) Tambahan tanggung jawab menulis counter di alur klaim/pembayaran.
