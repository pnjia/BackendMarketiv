# ADR-006 — Gunakan Zustand untuk State Management, Bukan Redux

## Status

Diterima.

## Context

- Aplikasi butuh state global (auth, campaign, wallet, notification) tetapi tim kecil dan ingin iterasi cepat.
- Redux menuntut banyak boilerplate (actions, reducers, store setup).

## Decision

Gunakan **Zustand** sebagai library state management.

## Consequences

- (+) Lebih ringan, API sederhana, tanpa boilerplate Redux.
- (+) Mudah dipakai bersama service layer dan hooks.
- (−) Ekosistem middleware/devtools tidak sekaya Redux untuk kebutuhan kompleks di masa depan.
