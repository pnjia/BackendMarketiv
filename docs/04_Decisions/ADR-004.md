# ADR-004 — Pisahkan `fraud_checks` dari `campaign_submissions`

## Status

Diterima.

## Context

- Submission membutuhkan hasil deteksi fraud (score, status, alasan).
- Phase 2 akan menambah jenis pengecekan: Computer Vision, Product Detection, Logo Detection.
- Menyimpan semua di dalam `campaign_submissions` akan membengkak dan sulit menyimpan histori banyak pengecekan.

## Decision

Pisahkan `fraud_checks` sebagai koleksi terpisah dengan relasi Submission 1 → N Fraud Checks. Field ringkas (mis. `fraudScore`, `fraudStatus`) tetap boleh ada di submission untuk query cepat.

## Consequences

- (+) Histori beberapa pengecekan fraud per submission dapat disimpan.
- (+) Phase 2 (CV/logo/product) bisa ditambahkan tanpa membengkakkan submission.
- (−) Perlu join/query tambahan untuk melihat detail histori fraud.
