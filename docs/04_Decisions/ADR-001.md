# ADR-001 — Gunakan Appwrite BaaS, Bukan Backend Custom

## Status

Diterima.

## Context

- MVP harus dibangun cepat oleh tim kecil (4 orang) dalam 12 minggu.
- Dibutuhkan auth, database, storage, realtime, dan logika server-side berbasis event.
- Membangun backend custom akan menambah beban infrastruktur dan waktu.

## Decision

Gunakan **Appwrite** sebagai BaaS untuk Auth, Database, Storage, Realtime, dan Functions. AI diakses lewat Appwrite Function wrapper.

## Consequences

- (+) Kecepatan pengembangan: fitur backend inti tersedia bawaan.
- (+) Realtime & event architecture didukung native (Functions + Events).
- (−) Bergantung pada kapabilitas & batasan Appwrite Cloud.
- Akses Appwrite tetap diabstraksi lewat service layer (lihat [ADR-002](ADR-002.md)).
