# ADR-003 — `orders` sebagai Aggregate Utama Alur Rate Card

## Status

Diterima.

## Context

- Alur Rate Card melibatkan offer, pembayaran, escrow, deliverable, dan revisi.
- Jika Offer mengelola Escrow langsung, tanggung jawab finansial dan status tersebar serta sulit dilacak.

## Decision

Jadikan `orders` sebagai aggregate utama. Offer tidak mengelola Escrow secara langsung. Alur:

```text
Offer → Order → Escrow → Deliverable → Revision → Completed
```

Order menjadi titik pusat status dan referensi finansial (escrow, deliverable, revisi).

## Consequences

- (+) Satu sumber kebenaran untuk status transaksi Rate Card.
- (+) Escrow dan deliverable terhubung jelas ke order.
- (−) Order harus dibuat sebelum escrow/deliverable diproses (langkah ekstra dari offer).
