# ADR-007 — Minimum Withdraw Dipasang sebagai Konstanta Sistem (= Rp50.000)

## Status

Diterima.

## Context

- Modul Payments butuh aturan batas bawah penarikan saldo (minimum withdraw) untuk menutup biaya transfer manual admin dan mencegah spam request kecil.
- Sebelum ADR ini, dokumentasi inkonsisten: workflow menyebut `MINIMUM_WITHDRAW` sebagai "konstanta sistem", sedangkan `Payments/70_Backend.md` menyebutnya "konfigurabel". Tidak ada koleksi `app_settings`, tidak ada endpoint admin, dan tidak ada UI admin untuk mengubah nilai.
- Tiga opsi yang dipertimbangkan:
  1. Hardcode sebagai konstanta di Appwrite Function.
  2. Environment variable Appwrite Function (bisa diubah tanpa redeploy kode, tapi tanpa UI admin).
  3. Koleksi `platform_settings` + halaman admin Settings (paling fleksibel, tapi butuh koleksi, API, dan UI baru).

## Decision

Minimum withdraw dipasang sebagai **konstanta sistem** dengan nilai tetap `Rp50.000` (`50.000`).

- Dideklarasikan sebagai konstanta di Appwrite Function `complete-withdrawal` (lokasi tunggal).
- Tidak ada koleksi database, environment variable, maupun endpoint admin untuk mengubah nilai.
- Pesan error ke user dan label UI menggunakan string literal `"Rp50.000"` — bukan variabel dinamis.
- Perubahan nilai di masa depan mensyaratkan update kode + redeploy Function.

## Consequences

- (+) Implementasi paling sederhana — satu konstanta, tidak ada koleksi/API/UI baru.
- (+) Tidak menambah surface area keamanan (tidak ada endpoint/settings yang bisa disalahgunakan).
- (+) Perilaku deterministik dan mudah diuji (nilai tidak berubah antar-deploy tanpa eksplisit).
- (−) Perubahan nilai butuh siklus deploy — bukan fitur operasional harian.
- (−) Tidak fleksibel untuk eksperimen A/B atau penyesuaian regional.
